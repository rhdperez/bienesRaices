PASOS GITHUB - VSC
Importante: la rama remota llamada origin se crea en un repositorio remoto.
La tecnología subyacente es Git y es lo que se maneja localemente,
la parte remota se maneja a traves de la herramienta Github.

*****https://www.youtube.com/watch?v=N-3IGVLdHOA (importante!!!!)

Local: init-->add-->Stating (prepara los cambios)--->commit-->Repositorio
Remoto: desde el Repositorio-->push al remoto / pull desde el remoto al Repo


1 Descargo Git : https://git-scm.com/download/win (muchas opciones de intalación)
2 Creo el repositorio LOCAL en la carpeta que quiera. O me paro donde quiero que este mi repositorio
3 Ejecuto "git init"-->lo que creará el "Repossitrio vacío: master" que es mi versión inicial. A esto le puedo crear varias ramas
las que quiera, cuidadndo el master, que lo trataría como la rama de producción.
4 ahora dentro de la carpeta donde estoy (que se creó el master), puedo chequear que estén mis archivos (en caso de que ya estén) con : "ls"
5 luego ejecuto un  "git add ." (que adhiere al Staging los archivos de la carpeta actual).
6 Se puede ver el estado "git status"
7 git commit -m "se añadieron nuevos archivos"-->para mandar al reposiotorio local
8 "git diff"-->se ven las diferencias añadidas...para revisar a nivel de código los cambios y otra info.
9 "git log"-->se ve un historial de los commit hechos y la info asociada.
10 Crear repositorio remoto en GitHub (publico), poner archivo "readme"
11 en VSC en el bash: añadir un repositorio remoto: "git remote add origin https:........(la URL del repositorio de GitHub)"
Esto creará un origen remoto
12 "git remote -v" se puede ver si se creó el fetch y push
13 Ahora antes de subir todo: hacer un pull
"git pull origin master"
14 Si sale el error "fatal: refusing to merge unrelated histories"-->quiere decir que tiene un repositorio que no es igual. Solución:
ejecutar: "git pull origin master --allow-unrelated-histories"-->se abre VIM
donde se puede comentar sobre el commit-->luego: "salir": escape :wq (windows quit). Y listooooo
15 clear....
16 ahora ya estaría: hacer "git push origin master"-->para mandar todo al repositorio GitHub.




Sincronizarlo VSC con el repositorio remoto: en VSC se inserta la url del repositorio remoto-->View/Comand Palet (Ctrl+shift+P), y poner "Git:Clone". Aquí se inserta
la URL del repositorio remoto (sacada de GitHub).

************si la carpeta master no esta en el lugar correcto*******************
La carpeta "master" en git se crea donde ejecutamos el "git init", y cada vez que nos paremos con nel Git bash en esta carpeta sale...(master), indicando
que es aquí la carpeta master. 
Ahora si la carpeta "master" no esta en el repositorio adecuado-->parase en ella ejecutar ls -a (para comprobar que esta el archivo .git) y:
remover la carpeta: "rm -rf .git".
Luego puedes crear un nuevo repositorio en la carpeta indicada, ubicándose en ella y 
ejecutando de nuevo el comando "git init" para inicializar un nuevo repositorio de Git en esa ubicación.o


Los cambios en el repositorio--->van a parar al "Stagin area" que es (local) antes de ir al repositorio remoto

(ver)-->https://www.youtube.com/watch?v=jGehuhFhtnE

La rama remota: es "origin".

La rama local es "master"

******----Publish to GitHub
Esto directamente publica la carpeta de trabajo actual en un repositorio remoto
en GitHub.
Una vez aceptado que VSC abra GitHub, pide que elijas el tipo de repositorio que quieres
ya inclutye el nombre del repositorio a crear igual al nombre de la carpeta local de trabajo.
De modo que una vez elegido el tipo de repo (private/public) aparecen todos los archivos de la carpeta
tildados por defecto. Aqui se elege si queremos publicar todos o destildar los que no se sincronizarán.


Una vez hecho todo....se pega en el git bash lo siguiente: (estando parado en el master)
git remote add origin https://github.com/rhdperez/bienesRaices.git
git branch -M master
git push -u origin master
